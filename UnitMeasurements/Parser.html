Overview

The UnitMeasurements::Parser class provides methods for parsing strings to extract quantity and associated unit. It can handle various formats, including complex numbers, scientific numbers, and rational numbers for the quantity.

.parse(string) ⇒ Array<Numeric, String|NilClass>

Parses a string to extract a quantity and its associated unit. This method first extracts a quantity and converts it to Float before returning it.

To get the correct parsed results, you must first normalize the string with Normalizer if using the parser standalone.

Examples:

Parsing string representing a complex number:

UnitMeasurements::Parser.parse("1+2i m")
=> [(1.0+2.0i), "m"]

Parsing string representing a rational number:

UnitMeasurements::Parser.parse("1/2 m")
=> [0.5, "m"]

Parsing string representing a mixed rational number:

UnitMeasurements::Parser.parse("2 1/2 km")
=> [2.5, "km"]

Parsing string representing a scientific number:

UnitMeasurements::Parser.parse("1e+2 km")
=> [100.0, "km"]

Parameters:

    The input string containing a quantity and an optional unit.

Returns:

    The parsed quantity and the unit associated with it (or nil if no unit is specified in the string).

Raises:

def parse(string)
  case string
  when COMPLEX_REGEX    then parse_complex(string)
  when SCIENTIFIC_REGEX then parse_scientific(string)
  when RATIONAL_REGEX   then parse_rational(string)
  else                       raise ParseError, string
  end
end

def parse_complex(string)
  real, imaginary, unit = string.match(COMPLEX_REGEX)&.captures
  quantity = Complex(real.to_f, imaginary.to_f)

  [quantity, unit]
end

def parse_rational(string)
  whole, _, numerator, denominator, unit = string.match(RATIONAL_REGEX)&.captures

  if numerator && denominator
    numerator = numerator.to_f + (denominator.to_f * whole.to_f)
    denominator = denominator.to_f
    quantity = Rational(numerator, denominator).to_f
  else
    quantity = whole.to_f
  end

  [quantity, unit]
end
